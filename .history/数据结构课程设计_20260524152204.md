# 七、核心算法设计（重点）

# 1. 最短路径算法

# Dijkstra算法

用途：

- 查询两个景点之间最短路径
    

时间复杂度：

O(n^2)

---

# Dijkstra核心思想

```txt
每次选择距离当前最近且未访问的节点，
不断更新其它节点距离，
直到找到终点。
```

---

# 2. 所有路径查询

# DFS（深度优先搜索）

用途：

- 查询两个景点之间所有可能路径
    

时间复杂度：

O(V+E)

---

# DFS核心思想

```txt
从起点不断向下搜索，
直到到达终点，
然后回溯继续寻找其它路径。
```

# 2. 为什么选择邻接矩阵

你可以写：

```txt
由于校园景点数量较少，
邻接矩阵实现简单，
查询边权速度快，
适合本系统。
```

---

# 3. 为什么选择Dijkstra

```txt
校园路径不存在负权值，
因此Dijkstra算法适用于校园导航系统。
```
