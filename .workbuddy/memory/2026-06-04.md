## C++ 校园导航系统 - 内网穿透 HTTP 服务器

**目标**: 将 C++ 后端通过 HTTP 暴露，配合内网穿透实现远程访问。

**方案**:
- 复制项目到 `server-http/` 目录，原项目不动
- 用纯 Winsock 手写单线程 HTTP 服务器（约200行），不依赖第三方库
- 原因：用户 MinGW 线程模型为 win32，不支持 `std::thread`，无法使用 cpp-httplib

**新增文件**:
- `server-http/server.cpp` - HTTP 服务器，包含 Dijkstra/DFS 算法 JSON 化
- `server-http/ui/map.html` - Canvas 交互式校园地图前端

**API 端点**:
| GET | 功能 |
|---|---|
| `/` | 返回交互式地图 |
| `/api/scenes` | 所有景点 JSON |
| `/api/scene?id=X` | 单个景点详情 |
| `/api/shortest?from=X&to=Y&type=Z` | 最短路径查询 |
| `/api/allpaths?from=X&to=Y` | 所有路径查询 |

**内网穿透使用方法**:
```bash
# 终端1
cd server-http && ./server.exe

# 终端2
ngrok http 8080  # 或 natapp
```

**编码修复**: data/scene.txt 和 data/road.txt 从 GBK 转为 UTF-8，解决浏览器显示乱码问题。
